## 楔子
今天(2022-10-24)把浏览器从谷歌迁移到的微软的Edge浏览器，在迁移密码的过程中，发现我总共300多个账号，却只用了9个密码。甚是惊讶。刚好想起来了上上个月写的半篇文章，现在把它补全电子化，贴到博客上。
> 为了防止歧路亡羊，建议看到一个觉得很适合自己的方法，就立马停止阅读此文，开始自己设计。

## 引言
现在(二十一世纪二十年代)很多媒体都在强调个人信息的重要性，强调密码的强度要高，提示说不要使用身份证、生日等容易破解的密码。但却并没有人教我们怎样设计一套自己的密码。

一直以来，人们都在最容易破解的密码之中徘徊。笔者曾经也是如此，四年级第一次去网吧申请的4399账号时随便打的`718293`竟一度成了后来的支付密码，手机号，姓名拼写等也都无脑的用来用去。可能是幸存者偏差的原因，所有的账户竟然一直没有被盗过号，也一直安心的用着，没有构想更安全的密码。
直到2019年高考报名时，班主任一再强调，“报名系统的密码要自己记住，强度要高，绝对不能让别人知道，以免别人篡改你的志愿”。对于这关乎一辈子命运的大事，笔者当然十分上心了。当时高考报名系统的密码要求是8位以上，包含大小写、数字以及符号，笔者一看，平常的垃圾密码肯定是用不成了，便苦思冥想了好几天，终于在复习化学时候，从元素周期表中找到了灵感——**利用化学元素与名字的谐音**(具体见下文)。当时设计完那个密码，真是提笔而立，为之踌躇满志，庖丁解牛后的感觉也大概是那样了。从那之后，慢慢掌握了一些密码设计的诀窍，逐渐地形成了一个个小窍门，而后这一系列的感兴终于在2022年末形成了文字。

## 密码的基本要求
本文主要是为了帮助普通人设计一套自己的密码体系，来更好、更方便地生活。本文所指代的密码就是一般的账户密码，而其他譬如凯撒密码、恩尼格码(摁你个妈)机、洛伦兹机、量子加密等加密体系由于极其难认，故此类密码学中的加密方法并不在本文的主要讨论范围内。为避免歧义，笔者决定将这种较短的、常用的、私人性质的密码称为——**密符**。(灵感来自周朝姜子牙用自己的鱼竿制作的密码——阴符)

笔者认为，这样一套密码体系有应该有两个最基本特点:强私人性与易记性。
私人性是首当其冲考虑的，因为每个人的经历不可能完全相同，不具有可重复性。哪怕你偶然读来一句诗，心念一动，根据物理上的平行宇宙假说，你就是进入了另一个平行宇宙了。可以说每个不经意的决定，每次环境潜移默化的影响，最终都成就了个体本身。生命看似是一条线，其实是万万条路中的一条，而这路上你独享的风景就是密符的重要来源。
其次，密码应当具备良好的易记性。其实设计密符就跟写诗一样，一句好的诗只消一瞥就永生难忘，一个好的密符亦是如此，设计完一看就能记住应当是好密符的重要特点。

接下来本文就以密符的设计为主题，从几个方面进行探讨:
## 密符的类型
一般而言，人们通常会遇到如下几类密符
1. 6位数字的支付密码、开机密码（亦即PIN）
2. 8位字符以上的字母、数字、符号组合密码（不做细分）
4. 9X9的手势密码
4. 指纹、人脸、瞳孔、声音等生物密码
5. 其他各种花哨的密码（比如黑莓android系列内置的图案移位密码，好像Win10上曾经也有过）

第一类可以归为第二类，第四类由于其绝对的私人性，可以说几乎不存在泄露、遗忘的风险，在此笔者将着重讨论第二、三类密符的设计。首先笔者将从设计方法和应用2个方面对各类密符的设计进行探讨

## 设计方法
为了实现密符的强私人性，我们需要一个专有的信息来源。这个来源必须是与我们生活直接相关的，如此才能保证我们可以更快更持久地记住这个密码，也自然的具有更高的强度。具体如下:

### 个人相关
#### 名字
因为现在的密码体系大多是英文。所以说，如果你有一个英文名的话最好不过，没有的话可以用拼音来代替。那么如何设计一个姓名相关的密码呢？下面以笔者的名字——**田欣洋**为例做个示范。

| 字  | 拼音 | 声调 | 笔画 |
|:---:|:----:|:----:|:----:|
| 田  | tian | 二声 | 五笔 |
| 欣  | xin  | 二声 | 八笔 |
| 洋  | yang | 二声 | 九笔 |


1. 拼音直用：`TianXinYang`
2. 拼音➕笔画：`5Tian8Xin9Yang`
3. 拼音➕音调：`Tian2Xin2Yang2`依照个人习惯取音调更好
4. 输入法转码
	1. 九键：`8426 946 9264`
		- 田：8426
		- 欣：946
		- 洋：9246
	1. 五笔：`LLLL RQWY IUDH`
		- 田：LLLL (田甲方框四车力)
		- 欣：RQWY (斤，⺈，人，丶)
		- 洋：IUDH(氵，丷，三，|)
1. 形合：`OHRIY`
	- 田→O
	- 欣→HR
	- 洋→iY
1. 意合：`filed love ocean`
2. 谐音：`tian si on`
	- 例子收集:`岳飞→ UFA`
1. 切音：`ti an xi yin yi ang`
2. 析字: `4shan -jin shui yang`
	- 田：四面山:4shan
	- 欣：欠斤两:-jin(减斤)
	- 洋：水边羊:shuiyang

经过这番准备，我们就可以发散思维来设计密符了:
1. 生辰八字：`86 410 410 610`
2. 生辰六字：`864040`

#### 地址
地址类密符可以从地区的简称、地理位置(相对方位、地标景点等)来进行映射，接下来以笔者的家乡大美南阳为例:
| 原始项目 |  隐含映射  | 输出映射 |
|:--------:|:----------:|:--------:|
|  河南省  |     豫     |    yu    |
|  南阳市  |     宛     |   Wan    |
|    南    | 伏牛山以南 |     South Mountain     |
|    阳      |    汉水以北        |    North River      |

这样映射以后很容易就可以得到一个不错的密符：`SMNR@YuWan`，含义是:**山南水北的豫之宛**。

但这样的映射还太简单，为了更具有私人性，再加上另一座笔者待过的城市乌鲁木齐来提高一下密符的强度
地址密符：`Sion@YuWan2XinU`，含义是：我Sion从南阳到新疆(Xin)乌鲁木齐(Urumqi)
#### 经历
普通人的一生会经历什么？出生，入学，升学，升学，升学，...，工作，结婚，生子(入学，升学...)，成就，变故，...，死亡

这些经历本身不单单是回首往事时的一句话，更是一首诗，用一辈子写下的诗篇。
1. 年份➕经历：`01born 19daxue 23biye`
2. 独特经历：`2019love@qwyc`
3. 独特经历：`2021@Web 2022@web3 2023@ai`

如此这般，不胜枚举。

## 图案密码设计
这里头的水太深，在此奉上[Zye](https://www.youtube.com/@DrZye)博主发布的唯一条视频：[What Is The Most Complicated Lock Pattern? - YouTube](https://youtu.be/PKjbBQ0PBCQ)
就我个人而言，技巧是**折回**:从中间点开始先连下面点，再折回到上面点来制造出不能一笔画的假象，如果还要说些什么的话，那就是：不要过于复杂，图案规律尽量有重复性，不对称性（不对称的安全性更高点）

## 应用体系
在了解了诸多的来源之后，我们便可以一展宏图了。我们可以根据密码的应用场景来设计密码。本文推荐的密码应用体系包含四类密码，它们的强度与应用性各有差异：

* 用于关键平台（比如微信、QQ、学信网等主要平台）
* 用于金钱相关的需要特别加密的平台（支付密码）
* 用一般社交兴趣网站（哔哩哔哩、知乎、微博等）
* 用于杂毛网站（用后即弃的，很少再用的）

## 管理
再好的密符，再好的应用体系，也绝非灵丹妙药，难以保证长久的安全，在此笔者对于个人密符的管理提出来一些建议
### 老生常谈
1. 定期更换
2. 不滥用
3. 使用密码管理软件
4. 使用密码生成器，搭配特定软件自动填写
### 密码保护
众所周知，密码学发展至今，已经出现了许多优质的加密方法。对称密码算法如**DES**,**AES**等，它们使用一个秘钥来进行加密，在秘钥安全的情况下，硬破解极为困难(废话)。还有现代最为广泛应用的非对称加密——公私钥体系，这类密码利用了数学上的陷门单向函数，有RSA，DHM(区块链上有很多应用)等。以及从公钥体系发展来的数字签名算法使用哈希函数来进行防篡改验证，算法有MD5, SHA系列,DIMON等。最后还有为了抵抗量子计算机而研制的抗量子加密算法，有Merkle(基于哈希)
、HFE(基于多变量)、NTRU系列(基于格，在2020年openssh 9.0将其作为默认加密算法)
。密码保护，顾名思义，就是用加密算法把设计好的密码进行加密，这样即便密码不小心泄露，别人也无法破解出来真正的密码。你甚至可以把加密好的密码，明文放到网络上，就以稍微落后的AES加密算法来讲，破解起来也要费好些功夫。当然，谁会对你没几个钱、没几个朋友的账户感兴趣呢？

下面以OpenSSL软件为基础，实操一下[^1]
#### 使用AES来加密文件
> 加密过程中会要求输入一个加密密码，重复输入两次即可完成对文件的加密。解密一个文件的时候会要求输入加密文件时设置的密码才能进行解密。


1. 使用 AES-256方法来加密一个文件 (data.zip 为原始文件，back.zip 为加密之后的文件)
```
# openssl enc -e -aes256 -in data.zip -out back.zip
```

2.  使用 openssl 解密一个文件 (back.zip 为加密的文件，data.zip 为解密之后的文件)

```
# openssl enc -d -aes256 -in back.zip -out data.zip
```

3. 解释
* enc：表示对文件进行对称加解密，
* -e： 表示对一个文件进行加密操作
* -d 表示对文件进行解密操作
* -aes256： 表示使用 aes256 算法进行操作
* -in：表示需要被加解密的文件
* -out： 表示加解密之后生成的新文件。
#### 使用RSA来加密文件

1. 首先需要使用 openssl 生成一个 2048 位的密钥 rsa.key 文件 (rsa.key 密钥文件中包含了私钥和公钥)

```
# openssl genrsa -out rsa.key 2048
```

2. 然后从 rsa.key 密钥文件中提取出公钥 pub.key

```
# openssl rsa -in rsa.key -pubout -out pub.key
```

3. 使用 pub.key 公钥加密一个文件 (data.zip 为原始文件，back.zip 为加密之后的文件)

```
# openssl rsautl -encrypt -inkey pub.key -pubin -in data.zip -out back.zip
```

4. 使用 rsa.key 私钥解密一个文件 (back.zip 为加密的文件，data.zip 为解密之后的文件)
